# 文档头添加 Skill

为项目markdown文档添加标准化的YAML frontmatter头，使doc-loader渐进式加载Agent能仅扫描头部即可完成相关性判断，无需grep全文。

## 何时使用

- **存量文档补充头**：项目中已有文档缺少frontmatter时，逐个添加
- **新文档创建时**：创建新设计文档时，自动添加标准头
- **文档头更新**：文档内容变更后，更新keywords或description

---

## 文档头规范

```yaml
---
type: 详设 | 接口契约 | 需求 | 架构 | 流程 | 测试 | 规范 | 说明
module: 模块名称 | 全局
description: 一句话描述文档内容与适用场景，前置关键触发词。Use when...
keywords: [关键词1, 关键词2, 关键词3]
dependencies: [依赖模块1, 依赖模块2]
version: "1.0"
---
```

### 字段定义

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `type` | 是 | enum | `详设`、`接口契约`、`需求`、`架构`、`流程`、`测试`、`规范`、`说明` |
| `module` | 是 | string | 所属业务模块名，跨模块填`全局` |
| `description` | 是 | string | 一句话，前置关键词，末尾附`Use when...`触发条件，风格同skill description |
| `keywords` | 否 | string[] | 触发关键词列表，中英文均可 |
| `dependencies` | 否 | string[] | 本文档依赖的其他模块名 |
| `version` | 否 | string | 文档版本号 |

### description书写规范

- **前置关键词**：把最重要的业务/技术关键词放在句子开头
- **末尾Use when**：附触发条件，告诉doc-loader何时推荐此文档
- **一句话**：不超过50字，信息密度最大化

**好的示例**：
```
description: 浏览器管理模块设计说明书，含生命周期管理、页面操作、context管理。Use when实现浏览器创建/销毁/页面导航功能时。
```

**不好的示例**：
```
description: 这个文档讲了一些关于浏览器的东西
```

## 操作流程

### 为单个文档添加头

1. **Read文档内容**：读取完整文档，理解其主题和范围
2. **提取信息**：
   - 从文档标题/内容推断 `type` 和 `module`
   - 从核心内容提炼 `keywords`（3-8个）
   - 从文档引用的其他模块提炼 `dependencies`
   - 综合上述信息撰写 `description`
3. **编辑文件**：在文档第一行前插入frontmatter块

### 批量为存量文档添加头

1. **glob扫描**：`glob **/*.md` 发现所有缺少头的文档
2. **逐个判断**：对每个文档read前10行，检查是否已有 `---` frontmatter
3. **按优先级处理**：先处理核心设计文档，再处理补充文档
4. **逐个添加**：按上述单文档流程逐个添加

### 判断文档是否已有头

文档的前3行如果以 `---` 开头，则已有frontmatter头，跳过或更新而非新增。

## type推断规则

当文档文件名不含明确类型标识时，根据内容特征推断：

| type | 推断依据 |
| --- | --- |
| `详设` | 文档含详细代码示例、类设计、方法签名、Story编号 |
| `接口契约` | 文档以API定义、请求/响应格式、参数说明为主体 |
| `需求` | 文档以业务场景、功能描述、验收标准为主体 |
| `架构` | 文档含模块关系图、分层架构、总体设计 |
| `流程` | 文档以时序图、交互流程、状态转换为主体 |
| `测试` | 文档含测试用例、测试场景、验证步骤 |
| `规范` | 文档以编码规范、命名约定、最佳实践为主体 |
| `说明` | README类项目介绍文档 |

---

## keywords提取规则

- 从文档标题中提取核心名词（2-3个）
- 从章节标题中提取技术术语（2-3个）
- 包含英文缩写和中文全称（如"SDK"、"MOON"、"HWCallback"）
- 优先选择需求描述中可能出现的关键词（面向需求匹配而非面向内容描述）

---

## dependencies提取规则

- 文档中引用的其他模块名（如"本模块依赖SDK模块"）
- import/include语句中引用的外部模块
- 接口契约中声明的外部接口所属模块
- 无明确依赖时，省略此字段
