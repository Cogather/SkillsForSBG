# 文档与代码一致性检查 Skill

## 触发时机
代码 commit 前

## 目标
检查设计文档与实际代码实现的一致性，确保文档准确反映代码设计意图。

## 检查流程

### 1. 获取变更文件
从 git diff 获取变更文件列表

### 2. 找到相关设计文档
根据变更文件路径，在 doc/ 目录下匹配相关文档：
- 文件名关键词匹配（Story、Feature、Module）
- 文档内容中引用的文件路径
- 文档目录结构映射

### 3. 提取文档关键描述
- 公开接口：方法名、参数、返回值
- 数据结构：字段名、类型、约束
- 常量配置：数值、名称、默认值
- 核心流程：关键步骤、条件分支
- 文件清单：新增/修改文件

### 4. 对比代码实现
使用 grep/read 检查代码中对应实现

### 5. 输出检查报告

| 文档描述 | 代码位置 | 状态 | 说明 |

状态标记：
- ✓ 一致
- ⚠ 合理差异（实现改进）
- ✗ 不一致（需更新）

## 检查原则

### 只检查关键项
- 公开接口定义
- 配置常量值
- 数据模型结构
- 核心流程逻辑
- 文件清单

### 合理差异
以下标记为 ⚠：
- 新增辅助接口（Mock 支持）
- 实现优化
- 代码风格调整
- 测试相关代码

### 不一致判定
以下标记为 ✗：
- 接口签名不匹配
- 常量值偏差
- 核心流程缺失
- 文档描述的文件不存在

## 输出格式
简洁表格，列出关键检查项，给出提交建议。
