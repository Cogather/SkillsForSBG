# 文档选择 Skill

根据用户需求描述，智能筛选出项目中最相关的文档，帮助AI快速定位设计依据。

## 何时使用

- **需求分析阶段**：用户提出新需求时，快速找到相关模块的设计文档
- **代码实现前**：需要查看设计文档了解模块细节
- **问题排查时**：根据问题描述找到相关模块的文档
- **文档编写时**：需要参考现有设计文档的格式和内容

## 文档选择流程

### 第一步：解析用户需求

**提取关键词**：
- 识别需求中的核心概念（告警、License、SNMP等）
- 提取技术术语（FM、CSP、高斯DB、选主等）
- 判断文档类型需求（需求文档、详设文档、接口文档）

**示例**：
```
用户输入："实现CSP告警汇总上报功能，需要参考设计文档"

关键词提取：
- CSP告警 ✅
- 汇总上报 ✅
- SNMP ✅
- 设计文档 ✅（需要详设文档）

匹配文档：
→ Story-5_GIDS SNMP告警上报软件详设.md
```

### 第二步：多关键词权重排序

**权重规则**：

| 权重 | 关键词类型 | 示例 |
| --- | --- | --- |
| **高（10分）** | 核心业务概念 | 告警、License、SNMP |
| **中（5分）** | 技术实现术语 | FM、CSP、选主、回调 |
| **低（2分）** | 通用描述词 | 实现、设计、文档 |

**文档得分计算**：
```python
def calculate_score(doc, keywords):
    score = 0
    for keyword in keywords:
        if keyword in doc.title:
            score += get_weight(keyword)
        if keyword in doc.content:
            score += get_weight(keyword) * 0.5
    return score
```

## 输出建议

### 何时提供完整文档内容

| 场景 | 提供方式 |
| --- | --- | --- |
| **代码实现前** | 提供完整文档内容 | 用户需要详细设计参考 |
| **问题排查时** | 提供相关章节内容 | 用户只需要特定部分 |
| **需求分析时** | 提供文档路径和摘要 | 用户可能需要多个文档 |

### 何时仅提供文档路径

| 场景 | 提供方式 |
| --- | --- | --- |
| **文档数量多** | 仅提供Top 3路径和摘要 | 避免输出过长 |
| **用户明确需求** | 提供单个文档路径 | 精准定位 |
| **初次探索** | 提供文档列表 | 用户需要自主选择 |

## 特殊场景处理

### 场景1：关键词不明确

**用户输入**：
```
"告警功能怎么实现的？"
```

**处理策略**：
1. 提供告警模块的所有相关文档（需求、流程、实现、接口）
2. 按文档类型排序：需求→流程→实现→接口
3. 添加说明："请根据需要查看不同类型的文档"

### 场景2：跨模块需求

**用户输入**：
```
"告警和License都需要查看设计文档"
```

**处理策略**：
1. 分别列出两个模块的文档
2. 按模块分组输出
3. 标注跨模块关联（如果有）

### 场景3：文档不存在

**用户输入**：
```
"查看话统上报的详设文档"
```

**处理策略**：
1. 搜索后发现无"话统上报"详设文档
2. 返回提示："当前项目中无话统上报详设文档"
3. 提供替代文档："可参考告警上报文档或需求文档中的话统章节"

## 文档类型说明

| 文档类型 | 文件特征 | 内容特点 | 使用场景 |
| --- | --- | --- | --- |
| **需求文档** | 文件名包含"需求" | SE视角，业务需求 | 理解业务背景 |
| **流程设计** | 文件名包含"流程" | 流程图、交互设计 | 理解整体流程 |
| **软件实现** | 文件名包含"软件实现" | 技术架构、代码框架 | 技术方案设计 |
| **详设文档** | 文件名包含"详设"、Story | 详细设计、代码示例 | 编码实现 |
| **接口文档** | 文件名包含"接口" | API定义、参数说明 | 接口对接 |

---

## 注意事项

1. **优先推荐详设文档**：代码实现时优先推荐Story详设文档
2. **需求文档作为补充**：理解业务意图时参考需求文档
3. **流程文档辅助理解**：整体架构理解时参考流程文档
4. **接口文档对接必需**：外部系统对接时必须提供接口文档
5. **AGENTS.md作为规范参考**：代码风格、编译规范参考AGENTS.md
